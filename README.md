# Siberian Wellness Parser

Локальный парсер каталога **[ru.siberianhealth.com](https://ru.siberianhealth.com/ru/shop/catalog/)** (Siberian Wellness) — собирает по каждому товару карточку (название, артикул, цена, баллы, состав, отзывы, ссылки на изображения и документы) и сохраняет её в виде одного текстового файла. Опционально файлы заливаются в Google Drive под аккаунтом владельца.

Проект задуман как **локальный сборщик базы данных для консультанта-нутрициолога**: каждый `.txt` — это полная карточка товара, удобная для чтения и поиска.

> **Платформа:** Windows 10/11. Все инструкции ниже — под Windows.
> **Готовность:** v0.1.0 (PR #1 замёржен в `main` на 5 мая 2026).

## Что делает парсер

1. **Discover** — открывает главную страницу Siberian Wellness, читает мега-меню, строит дерево из ~250 подкатегорий внутри 8 разделов.
2. **UI** — локальный веб-интерфейс на http://127.0.0.1:8765/, в котором пользователь галочками отмечает нужные подкатегории, ставит лимит товаров на категорию, включает/выключает Google Drive, указывает локальную папку.
3. **Parse** — для каждого выбранного товара открывает карточку через Playwright, кликает по вкладкам «Состав» и «Отзывы», проходит пагинацию отзывов, собирает все нужные поля (см. ниже).
4. **Save** — пишет один файл `<Название товара>.txt` в указанную локальную папку, и/или загружает тот же текст в Google Drive в папку `SiberianHealthParser/<категория>/<подкатегория>/`.

Парсер **не скачивает бинарные файлы** (картинки и PDF). Они остаются ссылками внутри `.txt`. Это сделано осознанно — нутрициолог открывает ссылку и работает с актуальной версией файла прямо с CDN Siberian Wellness.

## Технические решения

| Решение | Почему |
|---|---|
| **Playwright + Chromium вместо `requests`/`httpx`** | Сайт `ru.siberianhealth.com` блокирует прямые HTTP-запросы (Cloudflare) и требует выполнения JavaScript для рендеринга карточки товара. Полноценный браузер — единственный надёжный способ. |
| **BeautifulSoup для парсинга HTML** | После того как Playwright получил готовый DOM, BeautifulSoup быстрее и проще для CSS-селекторов, чем парсинг через сам Playwright. |
| **Один `.txt` на товар (а не папка с info.txt + images/ + documents/)** | Целевой пользователь — нутрициолог, который читает файлы из Drive. Один плоский `.txt` со всеми ссылками удобнее для поиска и не плодит папки. Бинарные файлы (PDF, JPG) живут на CDN Siberian Wellness и достаются по клику из текста. |
| **Tab pane `.im21--product-detail__tab-content` для описания** | На страницах с картинками вместо текста (например 500572 «3D Bone Vegan Cube») старый селектор `.im21--product-detail__about` возвращает только «Документы и материалы», а полный текст «О продукте» лежит в активной вкладке tab-content. Используем её первой, с fallback на старые селекторы. |
| **Парсинг блоков `.im21--product-options__option` парами «title/value»** | На сайте «Артикул» и «Количество в упаковке» лежат в одинаковых DOM-блоках, отличающихся только текстом заголовка. Делаем один проход по всем `__option`, читаем title-текст, и распределяем по нужным полям. |
| **Google Drive scope `drive.file`** | Самый узкий scope — приложение видит и редактирует **только** файлы, которые само создало. Не имеет доступа к остальным файлам в Drive пользователя. Это снимает большинство вопросов безопасности. |
| **OAuth 2.0 desktop client (а не service account)** | Сервис-аккаунты в Google не имеют собственного Drive (только Workspace shared drives), что усложняет настройку. Desktop OAuth позволяет напрямую заливать в личный Drive обычного аккаунта. |
| **OAuth client в режиме «Testing»** | Не требует прохождения Google-верификации (которая занимает недели и нужна только публичным приложениям). Минус: токен надо обновлять примерно раз в неделю и приложение надо явно добавить пользователю в Test users в Google Cloud Console. |
| **SQLite для state.db** | Лёгкое решение для resumable runs (если процесс упал на 100-м товаре, при следующем запуске продолжит с 101-го) и кеширования ID папок Drive. Без отдельного сервера БД. |
| **FastAPI + ванильный JS для UI** | Один процесс с веб-сервером, никакого `npm`/Node.js на стороне пользователя. JS без фреймворков — UI простой, обновлять зависимости легче. |
| **Таймаут страницы товара 90 с + 1 ретрай** | Реальные продакшен-наблюдения: некоторые товары (типа 500952 Pulse Box) иногда отдаются медленно из-за CDN. 45 с не хватало. 90 с + один ретрай через 2 с снимают почти все ложные таймауты, при этом один битый товар не валит всю категорию (ошибка пишется в лог, парсер идёт дальше). |
| **Имена файлов через `safe_folder_name()`** | Windows запрещает в именах файлов символы `\ / : * ? " < > \|` и зарезервированные имена (`CON`, `PRN`, и т.д.). Все названия товаров проходят через нормализацию: запрещённые символы заменяются на `-`, длина обрезается до 80, пустые имена становятся `untitled`. |
| **Логи через WebSocket в UI** | Пользователь видит прогресс в реальном времени, может нажать «Отмена» — парсер аккуратно завершит текущий товар и остановится (state.db сохранит точку прерывания). |

### Ключевые модули

- **`src/sibparser/site.py`** — мозг парсера. `Catalog` обходит каталог, `ProductScraper` парсит карточку, `_parse_product_html()` — чистая функция (без Playwright, тестируется без браузера) которая принимает HTML и возвращает `ProductCard`. `product_to_info_text(card)` рендерит карточку в финальный `.txt`.
- **`src/sibparser/runner.py`** — соединяет всё вместе. `Runner.discover_tree()` строит дерево категорий, `Runner.run(RunRequest)` обходит выбранные категории, для каждого товара вызывает `_handle_product()` который пишет `.txt` локально и/или в Drive.
- **`src/sibparser/drive.py`** — OAuth flow (`load_credentials` → если нет `token.json`, открывает браузер для consent), `DriveClient.ensure_path([категории])` создаёт цепочку папок и кеширует ID в `state.db`, `DriveClient.upload_text(name, text, parent_id)` заливает файл.
- **`src/sibparser/server.py`** — FastAPI поверх `Runner`. WebSocket `/ws/log` стримит `ProgressEvent` в UI.
- **`src/sibparser/config.py`** — единственный источник settings. Все пути в коде проходят через `get_settings()`.

## Установка с нуля (Windows)

> Эти шаги нужно сделать **один раз** при первой установке. Если у тебя уже всё настроено — сразу к разделу [Ежедневный запуск](#ежедневный-запуск).

### 1. Установить Python 3.12

1. Скачай Python 3.12 с https://www.python.org/downloads/windows/
2. При установке **обязательно** поставь галочку **«Add python.exe to PATH»**
3. Проверь в новом окне cmd:
   ```cmd
   py -3.12 --version
   ```
   Должно вывести `Python 3.12.x`.

### 2. Установить Git

1. Скачай Git с https://git-scm.com/download/win
2. Установка — все настройки по умолчанию
3. Проверь:
   ```cmd
   git --version
   ```

### 3. Клонировать репозиторий

> ⚠️ **Не клонируй в `C:\Users\Alex\Documents\`** — эта папка часто синхронизируется с OneDrive и/или защищена «Controlled folder access», что может блокировать запись `token.json` и других файлов парсера.

Рекомендуемое место — отдельная папка вне Documents:

```cmd
cd C:\Users\Alex
git clone https://github.com/magnitos93/siberianhealth-parser.git
cd siberianhealth-parser
```

Получится папка `C:\Users\Alex\siberianhealth-parser\` со всем кодом.

### 4. Создать виртуальное окружение и установить зависимости

```cmd
py -3.12 -m venv .venv
.venv\Scripts\activate
pip install --upgrade pip
pip install -e .
playwright install chromium
```

Последняя команда скачает headless Chromium (~150 МБ), которым парсер ходит на сайт.

### 5. Проверить что запускается

Двойной клик на **`run.bat`**. Должен открыться cmd c сообщением `Открой http://127.0.0.1:8765/` и сам открыться браузер на этом адресе.

Если открылся UI — установка завершена. Дальше — настройка Google Drive.

## Настройка Google Drive (OAuth)

Этот раздел тоже **разовый**. Файлы `credentials.json` и `token.json` после первой настройки используются автоматически.

### A. Создать OAuth-клиент в Google Cloud

1. **Создать проект** — https://console.cloud.google.com/projectcreate
   - Project name: `Siberian Parser` (или любое)
   - Location: No organization
   - **Create**, дождись «Project created», нажми **Select Project**

2. **Включить Drive API** — https://console.cloud.google.com/apis/library/drive.googleapis.com
   - Убедись, что в селекторе сверху выбран твой проект
   - Синяя кнопка **ENABLE**

3. **Настроить OAuth consent screen** — https://console.cloud.google.com/auth/overview
   - **GET STARTED**
   - **App name:** `Siberian Parser`
   - **User support email:** твой email
   - **Audience:** **External** → **Next**
   - **Contact information:** твой email → **Next**
   - Галочка «I agree…» → **Continue** → **Create**

4. **Добавить себя в Test users** — https://console.cloud.google.com/auth/audience
   - В разделе **Test users** жми **+ ADD USERS**
   - Введи свой email **в точности как ты заходишь в Google** (даже если ты владелец проекта — без этого Google заблокирует доступ с ошибкой `403 access_denied`)
   - **SAVE**

5. **Создать OAuth Client ID** — https://console.cloud.google.com/auth/clients
   - **+ CREATE CLIENT**
   - **Application type:** **Desktop app** (важно — не Web app!)
   - **Name:** `sibparser-desktop`
   - **Create**

6. **Скачать JSON** ★ КРИТИЧНО
   - В появившейся модалке **«OAuth client created»** внизу будет кнопка **⬇ DOWNLOAD JSON**
   - **Жми её сразу**, не закрывая модалку
   - Сохранится файл `client_secret_<номер>-<буквы>.apps.googleusercontent.com.json`
   - **Если ты закроешь модалку без скачивания — JSON уже не будет доступен**: Google запретил скачивание для уже созданных клиентов. Тогда придётся удалить клиент и создать заново.
   - **Не открывай файл в Блокноте** — Блокнот добавляет невидимый BOM, JSON ломается с ошибкой `Extra data: line 2 column 1 (char N)`.

### B. Положить credentials.json в проект

1. Открой Проводник в `C:\Users\Alex\siberianhealth-parser\`
2. Перетащи скачанный JSON в папку проекта
3. Переименуй ровно в **`credentials.json`** (через F2)
4. В Проводнике включи **Вид → ☑ Расширения имён файлов**, чтобы убедиться, что файл называется ровно `credentials.json`, а не `credentials.json.json`

### C. Войти в Google в UI парсера

1. Запусти `run.bat`
2. В UI парсера, секция **«1. Google Drive»** — кнопка **«Войти в Google»**
3. Откроется новая вкладка Google → выбери свой email
4. **Жёлтый экран «App isn't verified»** — это нормально, ты сам разработчик:
   - **Advanced** (или «Дополнительно»)
   - **Go to Siberian Parser (unsafe)**
5. Согласись с правом «See, edit, create only your own files» → **Continue**
6. Появится **«Authentication flow has completed»** — закрывай вкладку
7. В UI парсера секция Drive должна показать **✓ авторизован** + твой email
8. В папке проекта появится файл **`token.json`** — больше логиниться не нужно

### Важно знать про OAuth

- **Срок токена:** для приложений в режиме Testing refresh token обновляется автоматически, но иногда (~раз в неделю) Google всё равно требует повторного входа. Если в UI парсера статус Drive стал «не авторизован» — снова жми «Войти в Google», займёт 30 секунд.
- **Inactive cleanup:** Google удаляет OAuth-клиент после **6 месяцев** простоя. Если планируешь делать перерывы — раз в пару месяцев запускай хоть один товар, чтобы клиент считался активным.
- **Если потеряешь credentials.json** — придётся **удалить клиента** в Google Cloud Console и создать заново. Скачать тот же JSON повторно нельзя.

## Ежедневный запуск

После того как `credentials.json` лежит в проекте и ты хоть раз авторизовался в Google:

1. Двойной клик на `run.bat` → откроется UI парсера на http://127.0.0.1:8765/
2. В UI:
   - Убедись, что **Google Drive** показывает «✓ авторизован» (если нет — жми «Войти в Google»)
   - Включи галочки **«Загружать в Google Drive»** и/или **«Сохранять локально»**
   - В поле **«Папка для локального сохранения»** впиши путь, например `C:\Users\Alex\Сибирское здоровье` (поле сохраняется в браузере, второй раз вводить не надо)
   - Опционально: галочка **«Открыть папку в Проводнике после завершения»**
   - Поставь **лимит товаров на категорию** (0 = без лимита; для пробного запуска поставь 3)
3. Жми **«Открыть каталог»** — подождёт пока Playwright прогрузит главную страницу и покажет дерево из 8 разделов
4. Раскрой нужные разделы, отметь галочками подкатегории
5. Жми **«Запустить выбранные категории»**
6. В нижнем логе пойдут сообщения вида:
   ```
   → Витамины и добавки/3D Cube
     найдено 3 товаров в «3D Cube»
     · 3D Flex Cube
       txt 3D Flex Cube.txt
       drive 3D Flex Cube.txt
     ...
   Готово!
   ```
7. После «Готово!» (если включена галочка) откроется Проводник на папке с результатами

### Прервать запуск

Жми кнопку **«Отмена»** в UI. Парсер дождёт текущий товар, закроет браузер и выпишет последнее состояние в `state.db`. При следующем запуске можно продолжить с того же места.

## Конфигурация (`.env` и UI)

Большинство настроек делается прямо в UI и сохраняется в браузере (localStorage). Но если нужно поменять что-то системное — есть `.env` файл в корне проекта.

Скопируй `.env.example` → `.env` и редактируй по необходимости:

```ini
# Путь к Google OAuth credentials (по умолчанию ./credentials.json)
SIBPARSER_GOOGLE_CREDENTIALS=./credentials.json

# Путь к кешу OAuth-токена (по умолчанию ./token.json)
SIBPARSER_GOOGLE_TOKEN=./token.json

# Имя корневой папки в Google Drive (по умолчанию SiberianHealthParser)
SIBPARSER_DRIVE_ROOT_FOLDER=SiberianHealthParser

# Дефолтная локальная папка (если в UI не указано иначе)
SIBPARSER_DOWNLOADS_DIR=./downloads

# Файл состояния SQLite
SIBPARSER_STATE_DB=./state.db

# Показывать окно браузера при парсинге (true/false)
# Полезно для дебага. Для прод-запуска можно поставить false (быстрее).
SIBPARSER_HEADFUL=true

# Задержка между запросами в секундах (защита от rate-limit)
SIBPARSER_REQUEST_DELAY=0.5

# Адрес и порт UI
SIBPARSER_HOST=127.0.0.1
SIBPARSER_PORT=8765
```

## CLI-команды (для скриптов)

Если нужно запустить парсер не из UI, а из cmd-скрипта (например, по расписанию):

```cmd
.venv\Scripts\activate

REM Открыть UI (то же что run.bat)
python -m sibparser serve

REM Распечатать дерево каталога без авторизации в Drive
python -m sibparser discover

REM Распарсить один товар по URL без Drive (только локально)
python -m sibparser run --product https://ru.siberianhealth.com/ru/shop/catalog/product/500572/ --no-upload

REM Распарсить категорию с лимитом
python -m sibparser run -c "Витамины и добавки/3D Cube" --limit 3
```

## Известные ограничения

- **Сайт может менять разметку.** При смене CSS-классов нужно обновлять селекторы в `src/sibparser/site.py`. Тесты упадут — это сигнал, что нужна правка.
- **Cloudflare иногда показывает капчу.** Тогда Playwright не пройдёт. В headful-режиме (по умолчанию) можно решить вручную в окне браузера; в headless — придётся подождать и перезапустить.
- **Google OAuth в режиме Testing.** Каждый новый пользователь должен быть явно добавлен в Test users (даже сам владелец проекта). Не подходит для распространения парсера на множество людей — для этого надо проходить Google verification (отдельная задача).
- **Один товар = один файл.** Если `state.db` удалён, парсер не помнит что уже скачано и при повторном запуске **перезапишет** все `.txt`. Это OK потому что данные те же, но трата времени и сетевых запросов.
- **Большие категории (>200 товаров)** идут долго (~3-5 минут на 100 товаров с задержкой 0.5 с). Для скорости можно понизить `SIBPARSER_REQUEST_DELAY` до 0.1, но это рискует rate-limit.

## Решение типовых проблем

### `Discover: Internal Server Error`

Cloudflare или сам сайт ответил 5xx. Просто нажми «Открыть каталог» ещё раз. Парсер автоматически делает 1 ретрай через 2 секунды; если оба фейла — нужен ручной повтор.

### `Page.goto: Timeout 90000ms exceeded`

Конкретный товар не загрузился за 90 секунд + 1 ретрай. Парсер пишет это в лог и идёт дальше — один битый товар не валит всю категорию. После запуска можно перезапустить эту категорию ещё раз — обычно ловится со второго раза.

### `Авторизация: Extra data: line 2 column 1 (char 7)`

`credentials.json` повреждён (обычно из-за Блокнота, добавившего BOM). Перекачай JSON из Google Cloud Console **не открывая в редакторах**, либо если кнопка Download уже недоступна — удали клиент и создай заново.

### `Авторизация: [Errno 13] Permission denied`

Windows запрещает создавать файл `token.json` в папке проекта. Причины:
- Папка под OneDrive с активной синхронизацией → перенеси проект в `C:\Users\<имя>\siberianhealth-parser\`
- Включён «Controlled folder access» (Защитник Windows) → отключи или добавь Python в разрешённые приложения

### `403 access_denied` при входе в Google

Email, под которым ты заходишь, не добавлен в **Test users** в Google Cloud Console. Иди https://console.cloud.google.com/auth/audience → Test users → добавь свой email, сохрани, подожди 2-3 минуты, попробуй снова.

### Папка `SiberianHealthParser` не появляется в Drive

Смотришь не там. По умолчанию открывается раздел **Projects** — это **не то место**. Нужно зайти в **My Drive** (третий пункт в левом меню), там в корне будет папка `SiberianHealthParser`.

### UI открывается, но «старая» версия

Запущен старый процесс парсера. Закрой все cmd с парсером (Ctrl+C или закрой окно), сделай `git pull` в папке проекта, запусти `run.bat` заново. В Chrome — Ctrl+F5 для полного обновления страницы UI.

- ❌ Поиск/фильтр товаров внутри `.txt` — это работа целевой системы (Drive поиск, Notion, и т.д.), не парсера.
