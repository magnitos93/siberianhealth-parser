# Siberian Health Parser

Настраиваемый браузерный парсер каталога [ru.siberianhealth.com](https://ru.siberianhealth.com/ru/shop/catalog/). Каждый товар сохраняется как один текстовый файл `<Название>.txt` в папке своей категории — формат рассчитан на использование в качестве базы данных для консультанта-нутрициолога.

## Что умеет

- **Иерархия:** `Категория → Подкатегория → <Название товара>.txt` — на Google Drive и на локальном диске создаётся идентичная структура (`SiberianHealthParser/<категория>/<подкатегория>/3D Bone Vegan Cube.txt`).
- **Содержимое файла товара:**
  - название, URL, артикул, серия, цена, баллы, количество в упаковке;
  - короткое описание (под H1) **и** полное «О продукте» (включая страницы с картинками вместо текста — парсер забирает текст из всей вкладки);
  - состав;
  - все отзывы со всех страниц пагинации;
  - **ссылки** на картинки товара (открываются с CDN Siberian Wellness);
  - **ссылки** на документы и сертификаты (PDF, открываются по клику).
- **Никакого скачивания бинарников** — нутрициолог открывает `.txt`, видит всё в одном месте, при необходимости открывает картинку или PDF одним кликом по ссылке.
- **Настройка:** локальный веб-UI (открывается в твоём браузере) — выбираешь нужные категории галочками или вставляешь ссылку на отдельный товар. Можно поставить лимит товаров на категорию.
- **Возобновление:** прогресс пишется в SQLite (`state.db`), прерванный запуск можно продолжить.
- **Устойчивость:** таймаут открытия страницы товара 90 секунд + 1 автоматический ретрай. Один зависший товар не валит всю категорию.

## Стек

- Python 3.11+
- [Playwright](https://playwright.dev/python/) (браузер Chromium) для парсинга
- [FastAPI](https://fastapi.tiangolo.com/) + WebSocket для UI и live-логов
- [google-api-python-client](https://github.com/googleapis/google-api-python-client) для Google Drive
- SQLite для состояния и дедупа

## Установка

```bash

```

## Настройка Google Drive

1. Зайди в [Google Cloud Console](https://console.cloud.google.com/), создай (или выбери) проект.
2. Включи API: **APIs & Services → Library → Google Drive API → Enable**.
3. Создай OAuth-клиент: **APIs & Services → Credentials → Create Credentials → OAuth client ID**. Тип — **Desktop app**.
4. Скачай JSON и положи в корень проекта как `credentials.json` (или укажи путь в `.env`).
5. На странице **OAuth consent screen** добавь себя в Test users (если приложение в режиме Testing).

При первом запуске (нажатии «Войти в Google» в UI) откроется окно для согласия. После этого в проект сохранится `token.json`, и больше логин не понадобится.

> **Scope:** парсер запрашивает только `drive.file` — он видит и редактирует **только те файлы и папки, которые сам создал**. Доступа ко всему твоему Диску нет.

## Запуск

### Веб-UI (рекомендую)

**Windows — двойным кликом:** в папке проекта дважды щёлкни `run.bat`. Он сам активирует окружение и запустит сервер. Откроется окно cmd, дальше открой http://127.0.0.1:8765/ в браузере.

**Из терминала (Win/Mac/Linux):**

```bash
sibparser serve
# или:  python -m sibparser serve
```

UI откроется в браузере по адресу `http://127.0.0.1:8765/`. Дальше:

1. **«Войти в Google»** — авторизация Drive (нужна, только если хочешь заливать на Диск).
2. **«Открыть каталог»** — Playwright откроет сайт, прочитает меню и покажет дерево категорий.
3. Расставь галочки на нужных категориях (или вставь URL одного товара в правое поле «Один товар»).
4. В разделе «Опции запуска» выбери куда сохранять:
   - **Загружать в Google Drive** — заливка на Диск с дедупом сертификатов.
   - **Сохранять локально** — реально скачивать картинки и PDF в папку на диск.
   - **Папка для локального сохранения** — путь (по умолчанию `downloads/`, можно вписать `D:\Парсинг` и т.п.).
   - **Открыть папку в Проводнике после завершения** — после `Готово!` откроется указанная папка.
5. **«Запустить выбранные категории»** — лог пишется live в WebSocket.

Опции локального сохранения **независимы**: можно включить только Drive, только локально, или обе — тогда каждый файл скачивается **один раз** и сохраняется в обе цели. Сертификаты дедуплицируются и локально (через жёсткие ссылки в `_shared/certificates/`), и на Диске (через Drive shortcuts).

### Из командной строки

```bash
# Распарсить один товар (без Drive)
sibparser run --product https://ru.siberianhealth.com/ru/shop/catalog/product/400273/ --no-upload

# Распарсить категорию с лимитом 5 товаров
sibparser run -c "Каталог/Чаи и напитки" --limit 5

# Просто посмотреть дерево
sibparser discover
```

## Конфигурация

Все настройки читаются из `.env` (см. `.env.example`):

| Переменная | По умолчанию | Что это |
|---|---|---|
| `SIBPARSER_GOOGLE_CREDENTIALS` | `./credentials.json` | OAuth client secret |
| `SIBPARSER_GOOGLE_TOKEN` | `./token.json` | Кеш refresh-токена |
| `SIBPARSER_DRIVE_ROOT_FOLDER` | `SiberianHealthParser` | Имя корневой папки на Диске |
| `SIBPARSER_DOWNLOADS_DIR` | `./downloads` | Локальная папка для `--no-upload` режима |
| `SIBPARSER_STATE_DB` | `./state.db` | Файл SQLite |
| `SIBPARSER_HEADFUL` | `true` | Показывать окно браузера или нет |
| `SIBPARSER_REQUEST_DELAY` | `0.5` | Пауза между страницами, сек |
| `SIBPARSER_HOST` / `SIBPARSER_PORT` | `127.0.0.1` / `8765` | Адрес локального UI |
